All-In-One-Deflicker

AU Kwan Wo

Introduction

    All-In-One-Deflicker is a general postprocessing framework that can remove different types of flicker from various videos, including videos from video capturing, processing, and generation.


Setting up

    A set up for the model is provided in a Google Colab notebook as a demonstration. Follow the instructions to create the environment for the tool to work. You can also set up locally for faster runtime. Note that Linux is required to run the model with conda installed for setting up and managing environments. Details can be found under the github page: https://github.com/ChenyangLEI/All-In-One-Deflicker


Prompt

    To use the deflicker directly, you would only need to input the media you want to process, which can either be a video file or a folder with images (as 'frames' of a video). Nevertheless, if the result is unsatisfactory, more advanced options could also be considered:

  • Iteration numbers: you can increase the iteration numbers to increase the number of times processing the video.
  • Optical flow loss weight: you can tune the optical_flow_coeff according to the intensity of flicker in your video
  • Downsample rate: downsample rate can be adjusted according to your own needs. It would affect the convergence and slightly influences the quality
  • Maximum number of frames: maximum_number_of_frames could be adjusted to limit the number of frames in the output. Please note that it is recommend to split long videos into several
    • shorter sequences as performance for longer videos is yet to be evaluated.

interesting findings

    While the output still has flicker for the limit testing on the extreme light flickering example, a better result could be observed in concat.mp4 generated by the model under the neural_filter folder. It is surprising to see the video in the middle has flickering issues completely eliminated despite the lowered resolution. In extreme cases like this, one can also consider checking the concat.mp4 in search for a optimal result.


Conclusion

    Flicking in video shooting is a very common problem that occurs when frame rate and shutter speed do not match the capture of fractions of light pulses(try shooting a video on computer screen). In the past, deflickering either required manually adjusting parameters in dedicated softwares or only produced satisfactory results on specfic types of deflicking, but such types of issues can now be easily solved in All-In-One-Deflicker wtih significant effects on processing old cartoons, AI-generated videos and video captures.
